OpenX_Wrapper help and instructions.


1. Load website.


- select either MM (Money Morning) or DR (Daily Reckoning)
- "user" indicates an optional stylesheet has been loaded
- press button "LOAD site", this will load that site's styles

"RESET form" button will clear all fields.

Optional stylesheet loader:
- you can provide a full url to an appropriate stylesheet.
- press the "load style" button when ready.


2. Site icon.


- after the chosen site's stylesheets have loaded that site's will icon appear.
- "HELP!!" opens this guide in a new window.


3. Device type.


- select either PU (pop up), SF (sidebar form), IF (inline form) or EOA (end of article)
- press the "load" button to load that openX device with placeholder content

"update" button will load the current device with any content entered into the left side form fields.
"clear" button will remove the selected device.


4. Process.


- when the device is ready for use in OpenX, press the "process" button
- this will open a new window with code suitable for cut and paste into OpenX
- the start point for copying is declared with: <!-- start of device here -->
- the end of copying is declared with: <!-- end device -->

n.b. it does not matter if those declarations are included, they will be ignored.


Entry Form.


- this form has fields for generating a given OpenX device, as per the selections made above.
- report cover expects an absolute URL, ie. http://www.somesite.com.au/someReportCover.jpg (or .png)
- the fields marked "w" and "h" are optional width and height values, in pixels (a site default exists)
- "set heading" refers to the main title of the device, usually appearing in large letters
- "set byline" refers to an optional, secondary introduction to the device content
- "set copy para1..." , etc, refer to the main paragraphs of text to be displayed.
n.b. clearing their form field(s), clears them from being displayed.


Entities and Log.


- the black box is a console log that will print messages for use in debug
- to help with correct appearance of text on a webpage, you must use HTML entities
- listed are some common ones currently in use with OpenX devices
- NOTE: with some exceptions, they require an opening tag and a closing tag

KEY:
<br />
line break, you can use multiple to get extra spaces between paragraphs.

<strong>
is used to make any text between the tags appear bold

<em>
is used for italics between tags

&hellip;
is used for ellipsis, if, however, you use full stops in a row they may not stay together…

&lsquo;
is for left side, opening quote ‘

&rsquo;
is for right side, closing quote and apostrophes if desired ’

&ndash;
is for normal – dash

&mdash;
is for long — dash


bullet point lists: (unordered means they appear in the order you write them)


<ul>
starts the (unordered) list

<li>
starts an item in the list

</li>
ends that item in the list

</ul>
closes the (unordered) list


example:
<ul>
  <li> item 1 </li>
  <li> item 2 </li>
  <li> item 3 </li>
</ul>
